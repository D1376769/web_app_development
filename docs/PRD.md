# 產品需求文件 (PRD) - 讀書筆記本系統

## 1. 專案概述
- **背景與動機**：學生在學習過程中經常面臨知識零散、複習效率低下的問題。傳統筆記難以有效追蹤學習盲點，也缺乏系統化的複習機制。本專案旨在建立一個結構化的讀書筆記系統，幫助學生將零散知識整理為可快速複習與長期記憶的知識庫，並結合定期回顧與自我測驗機制，提升理解深度與學習效率。
- **目標用戶**：學生。
- **核心價值主張**：透過雙層筆記結構、標籤索引與間隔重複測驗，提供一站式的知識管理與高效學習解決方案，幫助學生克服遺忘曲線。

## 2. 功能需求
1. **雙層筆記結構（摘要＋延伸）**
   - *作為學生，我希望能夠建立包含「核心摘要」與「延伸思考」的雙層筆記，以便在快速複習重點的同時，也能深入了解細節與記錄個人見解。*
2. **標籤＋主題交叉索引**
   - *作為學生，我希望能夠為筆記加上多個標籤並設定主題，以便在未來能透過交叉索引快速找到跨領域或特定主題的相關知識點。*
3. **錯題／盲點追蹤系統**
   - *作為學生，我希望系統能記錄我測驗時答錯或標記為不熟悉的知識點，以便集中時間與精力針對學習盲點進行補強。*
4. **間隔重複（Spaced Repetition）**
   - *作為學生，我希望系統能根據我對各個筆記的熟悉程度自動排程複習計畫，以便在遺忘的關鍵時刻提醒我複習，有效轉化為長期記憶。*
5. **自我測驗模式（主動回憶）**
   - *作為學生，我希望能夠將筆記轉換為自我測驗題（如閃卡或問答），以便透過主動回憶的方式自我檢驗學習成效。*

## 3. 非功能需求
- **技術限制**：後端採用 Flask 框架，前端模板使用 Jinja2，資料庫使用 SQLite。
- **效能考量**：頁面載入迅速，操作順暢，查詢筆記及載入測驗題目不卡頓。
- **安全考量**：保護使用者的筆記資料，避免未經授權的存取（若支援多用戶）。
- **介面設計**：提供直覺、專注且易於閱讀的使用者介面。

## 4. MVP 範圍
- **Must Have（必須有）**
  - 雙層筆記的新增、檢視、編輯與刪除 (CRUD)。
  - 標籤與主題的基本分類機制。
  - 基礎的自我測驗模式（如閃卡翻轉）。
- **Should Have（應該有）**
  - 錯題與盲點追蹤（標記不熟的筆記）。
  - 間隔重複的演算法（簡易版，例如每日推薦複習清單）。
- **Nice to Have（可以有）**
  - 學習數據統計（如每日複習量、錯題率圖表）。
  - 支援 Markdown 或富文本編輯。
